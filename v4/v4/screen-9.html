Limitations of this approach

1

No holistic visibility of all features at once

The linear chat flow only shows one feature at a time. The channel manager can't see the full picture — which features are done, which are pending, how they relate to each other. There's no dashboard or bird's-eye view.

Impact: User can't quickly check "what did I set for Deal Rules?" without scrolling back through conversation history. No persistent state reference panel.
Design Question

Should there be a persistent sidebar or summary card that shows all 6 features and their status at all times?

2

External navigation breaks the flow

Some features may require the user to leave this page — e.g., uploading a CSV of partner data, configuring SSO in a separate admin panel, or approving something in another Salesforce tab. Once they leave, the conversational context is lost.

Impact: No mechanism to bring the user back to exactly where they left off. Browser back button loses state. No deep-link back to "Feature 4, step 2."
Design Question

Can we persist a "Return to setup" notification/banner? Should the agent send a follow-up prompt when it detects the user completed the external action?

3

Abandoned sessions have no recovery path

If the user closes the browser, gets interrupted mid-setup, or simply runs out of time — there's no guaranteed way to resume. The chat history might persist, but the interactive cards with editable fields won't regenerate in the same state.

Impact: User returns to a dead conversation. They have to say "continue where I left off" and hope the agent can reconstruct state. Saved features are fine — but in-progress ones are lost.
Design Question

Should we auto-save drafts? Show a "Resume setup" card on the Ask Home page? Persist partial config to the backend before the user hits "Save"?
